# PYRO — голосовой ассистент на локальных LLM

**PYRO** – это полностью локальный голосовой ассистент, который понимает речь и текст, поддерживает диалог на русском языке, выполняет команды и отвечает синтезированной речью. Всё работает на вашем компьютере без подключения к облачным API.

---

## Возможности

- 🎤 **Голосовое управление** – активация по ключевому слову **«слушай»**, распознавание речи через Vosk.
- 🧠 **Диалог на естественном языке** – генерация ответов с помощью модели `llama3.1:8b` (Ollama).
- 🗣️ **Озвучивание ответов** – синтез речи (Silero TTS) с выбором голоса.
- ⚙️ **Специальные команды** – выключение, перезагрузка, запись голоса по требованию, открытие сайтов.
- ⌨️ **Текстовый ввод** – удобно для отладки или если микрофон недоступен.
- 🔄 **Автоматический запуск** – сервер Ollama стартует при необходимости.
- 📚 **Контекстный диалог** – ассистент помнит последние 20 обменов.

---

## Системные требования

- **ОС**: Windows (10/11), Linux, macOS.
- **Python**: 3.8 – 3.11.
- **Аппаратное обеспечение**:
  - Минимум 8 ГБ ОЗУ (рекомендуется 16 ГБ для комфортной работы с моделью 8B).
  - Микрофон и динамики/наушники.
  - GPU с CUDA (опционально, ускоряет TTS).
- **Предварительно установленные компоненты**:
  - [Ollama](https://ollama.com) с моделями `llama3.1:8b` и `qwen2.5:1.5b`.
  - Модель Vosk для русского языка (например, `vosk-model-small-ru-0.22`).

## Установка и настройка

1. **Клонируйте репозиторий** или скачайте все файлы проекта.

2. **Установите Python-зависимости**:
   ```bash
   pip install ollama vosk pyaudio requests soundfile sounddevice numpy torch
   ```
   > На Windows для `pyaudio` может потребоваться бинарный wheel (например, установите через `pipwin`).

3. **Скачайте Vosk-модель**:
   - Загрузите [vosk-model-small-ru-0.22](https://alphacephei.com/vosk/models) и распакуйте в папку `../vosk-model-small-ru-0.22/` относительно корня проекта (путь можно изменить в коде).

4. **Установите и запустите Ollama**:
   - Следуйте инструкциям на [официальном сайте](https://ollama.com).
   - Скачайте нужные модели:
     ```bash
     ollama pull llama3.1:8b
     ollama pull qwen2.5:1.5b
     ```

5. **Проверьте наличие файлов**:
   - `russian_prompt.py` – содержит системный промпт для диалоговой модели (создайте его самостоятельно, пример в документации).
   - `tts_engine.py` – модуль синтеза речи (уже есть в проекте).
   - `research.py` – содержит функцию `web_search` для открытия сайтов.

6. **Запустите ассистента**:
   ```bash
   python LLM.py
   ```

## Использование

### Голосовой режим
1. Произнесите ключевое слово **«слушай»**.
2. После короткой паузы ассистент активируется и начнёт запись вашей команды или вопроса.
3. Скажите запрос (например, *«какая погода?»* или *«открой ютуб»*).
4. Ассистент распознает речь, обработает запрос и ответит голосом.

### Текстовый режим (консоль)
Просто введите текст в терминале и нажмите Enter – ассистент ответит без активации ключевым словом. Для выхода введите `выход`, `exit` или `quit`.

## Поддерживаемые команды

Ассистент распознаёт специальные команды через отдельную модель `qwen2.5:1.5b`:

| Команда       | Пример фразы               | Действие                                                                 |
|---------------|----------------------------|--------------------------------------------------------------------------|
| `shutdown`    | «выключись», «заверши работу» | Завершает работу ассистента.                                             |
| `restart`     | «перезагрузи систему»      | Перезапускает сервер Ollama, очищает историю диалога и приветствует.   |
| `record`      | «запиши», «слушай» (без активации) | Начинает запись голоса и распознаёт сказанное как новую команду.        |
| `web`         | «открой ютуб», «наса»      | Открывает указанный сайт в браузере (домен из аргумента).                |
| `none`        | (любая другая фраза)       | Передаёт запрос в диалоговую модель `llama3.1:8b` для генерации ответа.  |

## Структура проекта

- **`LLM.py`** – главный скрипт: инициализация, основной цикл, интеграция всех модулей.
- **`tts_engine.py`** – синтез речи на базе Silero TTS (функции `speak`, `text_to_speech`, `play_audio`, `save_audio`).
- **`functionReading.py`** – парсер команд через модель `qwen2.5:1.5b` (класс `CommandParser`).
- **`research.py`** – открытие веб-сайтов (`web_search`).
- **`russian_prompt.py`** – системный промпт для диалоговой модели.

## Конфигурация

- **Смена диалоговой модели** – измените `self.model` в классе `PyroQwen` (например, на `"llama3.1:70b"`).
- **Ключевое слово активации** – передайте другой параметр `keyword` при вызове `background_voice_listener()` в `main()`.
- **Голос TTS** – в `speak()` или `text_to_speech()` можно указать другого диктора (список: `"eugene"`, `"baya"`, `"kseniya"`, `"xenia"` и др.).
- **Путь к модели Vosk** – измените `model_path` при создании `voiceRecorder`.

## Ограничения и известные проблемы

- На Windows сервер Ollama запускается с флагом `CREATE_NO_WINDOW` – для Linux/macOS потребуется адаптация (уберите этот флаг в коде).
- Маленькая модель Vosk может давать ошибки на шумной речи – используйте модель большего размера, если позволяет ОЗУ.
- Во время выполнения команды `record` могут возникнуть конфликты с фоновым прослушиванием (микрофон занят). Рекомендуется не использовать `record` в голосовом режиме.
- История диалога не сохраняется между сессиями.
- Требуется стабильный запущенный сервер Ollama – при первом запуске может потребоваться время на загрузку модели.
